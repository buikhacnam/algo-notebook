## Description

Given an array of functions <code>[f<span font-size: 10.8333}>1</span>, f<sub>2</sub>, f<sub>3</sub>, ..., f<sub>n</sub>]</code>, return a new function <code>fn</code> that is the function composition of the array of functions.

The function composition of <code>[f(x), g(x), h(x)]</code> is <code>fn(x) = f(g(h(x)))</code>.

The function composition of an empty list of functions is the identity function <code>f(x) = x</code>.

You may assume each function in the array accepts one integer as input and returns one integer as output.

Example 1:

<br />
<pre className="overflow-x-auto">
	Input: functions = [x =&gt; x + 1, x =&gt; x * x, x =&gt; 2 * x], x = 4
	Output: 65 Explanation: Evaluating from right to left ... Starting with x = 4.
	2 * (4) = 8 (8) * (8) = 64 (64) + 1 = 65
</pre>

Example 2:

<br />
<pre className="overflow-x-auto">
	Input: functions = [x =&gt; 10 * x, x =&gt; 10 * x, x =&gt; 10 * x], x = 1
	Output: 1000 Explanation: Evaluating from right to left ... 10 * (1) = 10 10 *
	(10) = 100 10 * (100) = 1000
</pre>

Example 3:

<br />
<pre className="overflow-x-auto">
	Input: functions = [], x = 42 Output: 42 Explanation: The composition of zero
	functions is the identity function
</pre>

Constraints:

    - <code><font face="monospace">-1000 &lt;= x &lt;= 1000</font></code>
    - <code><font face="monospace">0 &lt;= functions.length &lt;= 1000</font></code>
    - <font face="monospace"><code>all functions accept and return a single integer</code></font>
