## Description
Given an array <code>arr</code> and a chunk size <code>size</code>, return a chunked array. A chunked array contains the original elements in <code>arr</code>, but consists of subarrays each of length <code>size</code>. The length of the last subarray may be less than <code>size</code> if <code>arr.length</code> is not evenly divisible by <code>size</code>.

You may assume the array is the output of <code>JSON.parse</code>. In other words, it is valid JSON.

Please solve it without using lodash&#39;s <code>_.chunk</code> function.

 
Example 1:

<br /><pre className="overflow-x-auto">
Input: arr = [1,2,3,4,5], size = 1
Output: [[1],[2],[3],[4],[5]]
Explanation: The arr has been split into subarrays each with 1 element.
</pre>

Example 2:

<br /><pre className="overflow-x-auto">
Input: arr = [1,9,6,3,2], size = 3
Output: [[1,9,6],[3,2]]
Explanation: The arr has been split into subarrays with 3 elements. However, only two elements are left for the 2nd subarray.
</pre>

Example 3:

<br /><pre className="overflow-x-auto">
Input: arr = [8,5,3,2,6], size = 6
Output: [[8,5,3,2,6]]
Explanation: Size is greater than arr.length thus all elements are in the first subarray.
</pre>

Example 4:

<br /><pre className="overflow-x-auto">
Input: arr = [], size = 1
Output: []
Explanation: There are no elements to be chunked so an empty array is returned.</pre>

 
Constraints:


	- <code>arr is a valid JSON array</code>
	- <code>2 &lt;= JSON.stringify(arr).length &lt;= 10<sup>5</sup></code>
	- <code>1 &lt;= size &lt;= arr.length + 1</code>
